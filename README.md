# Онлайн-лист персонажа для DnD 5e

Командный курсовой проект — система для ведения листов персонажей Dungeons & Dragons 5-го издания в реальном времени. Состоит из трёх компонентов: бэкенд-сервиса, веб-приложения для мастера и мобильного приложения для игроков.

## Возможности системы

- Полноценный цифровой лист персонажа DnD 5e — характеристики, боевые параметры, навыки, спасброски, атаки, заклинания, снаряжение, внешность, предыстория
- Игровые комнаты с уникальными кодами подключения; поддержка публичных и приватных сессий
- Синхронизация изменений между всеми участниками комнаты в реальном времени через WebSocket
- Бросок кубиков (d4–d100) в публичном и скрытом режимах с трансляцией результатов
- Магический шар — 20 классических ответов
- Шаблоны персонажей на уровне аккаунта для повторного использования
- Журнал событий сессии (броски, подключения, изменения персонажей)
- Управление участниками мастером: кик и завершение сессии 
- Подсчёт статистики побед/поражений

## Компонент 1: Бэкенд

### Стек
- **C# / ASP.NET Core 8.0** — REST API и SignalR-хаб
- **Entity Framework Core 8.0 + Npgsql** — Code-First, три изолированные схемы PostgreSQL (`identity`, `rooms`, `characters`)
- **PostgreSQL 15** — основное хранилище данных
- **FluentValidation** — валидация входных данных
- **BCrypt.Net** — хэширование паролей
- **JWT Bearer** — аутентификация; in-memory blacklist для отзыва токенов
- **Docker / Docker Compose** — контейнеризация, миграции применяются автоматически при запуске

### Архитектура
Монолит с модульной структурой:

```
backend/
├── DnD_backend/     # Точка входа, DI, middleware-конвейер
├── Identity/        # Аутентификация, профиль пользователя
├── Rooms/           # Комнаты, участники, кубики, журнал событий
├── Characters/      # Листы персонажей, атаки, заклинания, шаблоны
├── RealTime/        # SignalR-хаб (/hubs/session), трекер подключений
└── Shared/          # JWT, обработка ошибок, доменные справочники DnD5
```

### Запуск

```bash
cd backend
docker compose up -d --build
```

| Адрес | Описание |
|-------|----------|
| http://localhost:5050 | REST API |
| http://localhost:5050/swagger | Документация Swagger UI |
| ws://localhost:5050/hubs/session | SignalR-хаб |

### Переменные окружения

| Переменная | Описание |
|-----------|----------|
| `ConnectionStrings__DefaultConnection` | Строка подключения к PostgreSQL |
| `Jwt__Key` | Ключ подписи JWT, минимум 32 символа |
| `Jwt__LifetimeMinutes` | Срок жизни токена (по умолчанию 1440 = 24 ч) |
| `Cors__AllowedOrigins` | JSON-массив допустимых CORS-источников |

### Основные эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/auth/register` | Регистрация |
| POST | `/auth/login` | Вход, получение JWT |
| POST | `/auth/logout` | Отзыв токена |
| POST | `/rooms` | Создание комнаты |
| POST | `/rooms/{code}/join` | Вход по коду |
| POST | `/rooms/{id}/finish` | Завершение сессии (только мастер) |
| GET | `/rooms/{id}/state` | Состояние комнаты |
| POST | `/rooms/{id}/dice/roll` | Бросок кубика |
| GET | `/rooms/{id}/events` | Журнал событий |
| POST | `/rooms/{id}/characters` | Создание персонажа |
| PATCH | `/rooms/{id}/characters/{cid}` | Обновление полей персонажа |
| DELETE | `/rooms/{id}/characters/{cid}` | Удаление (архивация) |
| POST | `/rooms/{id}/characters/{cid}/attacks` | Добавить атаку |
| PUT | `/rooms/{id}/characters/{cid}/attacks/{aid}` | Обновить атаку |
| DELETE | `/rooms/{id}/characters/{cid}/attacks/{aid}` | Удалить атаку |
| POST | `/rooms/{id}/characters/{cid}/spells` | Добавить заклинание |

| DELETE | `/rooms/{id}/characters/{cid}/spells/{sid}` | Удалить заклинание |
| GET | `/characters` | Шаблоны персонажей |
| POST | `/characters` | Создать шаблон |

Полная документация — в Swagger UI.

### События SignalR

Подключение: `wss://host/hubs/session?access_token=<jwt>`, затем вызов `JoinRoom(roomId)`.

| Событие | Когда отправляется |
|---------|--------------------|
| `character.updated` | Персонаж создан, изменён или удалён; payload содержит полный актуальный объект |
| `dice.rolled` | Результат броска (публичный или скрытый) |
| `participant.joined` | Игрок подключился к комнате |
| `participant.left` | Игрок вышел или был исключён |
| `room.updated` | Изменился статус комнаты |

## Компонент 2: Фронтенд (веб-приложение мастера)

### Стек
- **React 18** — UI
- **React Router DOM 6** — маршрутизация
- **Zustand** — управление состоянием
- **Axios** — HTTP-запросы к API
- **@microsoft/signalr 8** — real-time обновления
- **React Hook Form + Yup** — формы и валидация

### Экраны

| Страница | Описание |
|---------|----------|
| Главная | Лендинг, навигация |
| Регистрация / Вход | Создание и авторизация аккаунта |
| Профиль | Данные аккаунта, статистика |
| Мои комнаты | Список сессий пользователя |
| Сессия | Основной игровой экран мастера |

**Игровой экран сессии** включает:
- Листы персонажей всех участников с редактированием в реальном времени (дебаунс 400 мс)
- Трекер инициативы
- Панель броска кубиков
- Список игроков с онлайн-статусом
- Журнал событий (броски, подключения, изменения)
- Модал завершения сессии

### Запуск

```bash
cd frontend
npm install
npm start
```

Приложение доступно по адресу **http://localhost:3000**.

## Компонент 3: Андроид (мобильное приложение игрока)

### Стек
- **Java 8** — язык разработки
- **Retrofit 2 + Gson** — REST-клиент
- **OkHttp 4 + LoggingInterceptor** — HTTP с логированием
- **Microsoft SignalR 8** — real-time обновления
- **Room 2** — локальная база данных (кэш персонажей и шаблонов)
- **RxJava 3** — реактивное программирование
- **AndroidX Navigation** — навигация между фрагментами
- **minSdk 27** (Android 8.1) / **targetSdk 34** (Android 14)

### Экраны

| Экран / Фрагмент | Описание |
|-----------------|----------|
| LoginActivity | Вход в аккаунт |
| RegisterActivity | Регистрация |
| GamesFragment | Список комнат пользователя |
| RoomInfoFragment | Информация о комнате, участники |
| CharactersFragment | Список персонажей |
| CharacterEditorFragment | Редактор листа персонажа |
| RoomCharacterPickerFragment | Выбор персонажа при входе в комнату |
| DiceFragment | Бросок кубиков |
| ProfileFragment | Профиль и статистика |

### Запуск

1. Открыть папку `android/` в **Android Studio**
2. Дождаться синхронизации Gradle
3. Запустить на эмуляторе (API 27+) или физическом устройстве

> Перед запуском убедитесь, что бэкенд запущен и доступен с устройства/эмулятора. Адрес API задаётся в конфигурации сетевого клиента.

```bash

# 3. Android — открыть в Android Studio и запустить
```
